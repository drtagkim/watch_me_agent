🛠️ 설치 및 설정 (Installation)

프로그램을 처음 사용하기 위한 필수 설정 과정입니다.

  1. 터미널(Terminal)을 열고 구동할 폴더로 이동합니다.
    cd /Users/tagg/dev/WatchPresentation
  2. 파이썬 가상환경(venv)을 생성하고 활성화합니다.
    python3 -m venv venv
    source venv/bin/activate
  3. 필요한 파이썬 패키지를 설치합니다.
    pip install -r requirements.txt
  4. .env 파일을 생성하고 발급받은 GEMINI_API_KEY를 입력합니다.
    echo "GEMINI_API_KEY=your_key_here" > .env

🚀 빠른 시작 (Quick Start - 권장)

프로그램을 실행하는 가장 간단하고 권장되는 방법입니다.

  1. 터미널(Terminal)을 엽니다.
  2. 프로젝트 폴더로 이동합니다.
    cd /Users/tagg/dev/WatchPresentation
  3. 실행 스크립트를 구동합니다.
    ./watch_run.sh
참고: 최초 실행 시 llama.cpp (Qwen3-VL-8B 모델) 서버가 반드시 127.0.0.1:8080 포트에서 실행 중이어야 합니다.

⚙️ 실행 시 설정 항목 (대화형)

./watch_run.sh 실행 시, 조교(Yoon Ina)가 두 가지 질문을 드립니다.

1. 분석 주기 (Chunk Duration) 설정

⏱️ 분석 주기를 몇 초 단위로 설정하시겠습니까? (기본값: 30):
  • 의미: 몇 초마다 끊어서 음성과 화면을 AI(Qwen)에게 분석 맡길지 결정합니다.
  • 입력 예시: 15, 60, 120 등 숫자로 입력 후 엔터.
  • 기본값: 그냥 엔터를 치면 무난한 30초 단위로 자동 설정됩니다.
  • Tip: 호흡이 긴 강연은 60초 이상, 즉각적인 반응이 필요한 회의는 15~30초를 추천합니다.

2. 동적 맞춤 분석 (Focus Area) 설정

🎯 이번 발표/회의에서 특별히 집중 분석할 키워드나 관점이 있나요? (엔터 입력 시 기본 모드):
  • 의미: AI가 "무엇을 집중적으로 볼지" 목표를 부여합니다. Gemini 모델이 이를 해석하여 Qwen에게 맞춤형 지시를 내립니다.
  • 입력 예시: 연구 방법의 한계점 찾기, 인공지능 윤리적 쟁점, 경쟁사와의 차별점
  • 기본값: 그냥 엔터를 치면 기본 학술/연구 발표 심사 모드(기록/요약/평가)로 작동합니다.

💻 파이썬 직접 실행 모드 (Advanced)

쉘 스크립트(watch_run.sh)를 거치지 않고 직접 세부 옵션을 설정하고 싶을 때 사용합니다.

  • 기본 실행: (30초 단위 분석, 기본 학술 모드)
    python watch_and_analyze.py
  • 분석 주기 시간 변경 (--chunk): (예: 60초 단위)
    python watch_and_analyze.py --chunk 60
  • 집중 분석 키워드 지정 (--focus):
    python watch_and_analyze.py --chunk 60 --focus "경쟁사와의 차별점 분석"

📂 생성되는 결과물 (Outputs)

분석이 시작되면 실시간으로 다음 파일들이 생성됩니다.

  • 실시간 마크다운 로그 (Realtime_Log_*.md): 매 국면마다 텍스트 변환(STT), 웹캠 스냅샷, Gemini의 영/한 질문, Qwen의 심층 분석 내용이 실시간 누적 기록됩니다.
  • 질문 모음 (Realtime_Log_*_question.md): 세션 중 파생된 핵심 질문(Micro/Macro)들만 따로 모아둔 파일입니다. 회의/발표 끝난 직후 Q&A 시간에 바로 활용하세요.
  • 최종 총괄 리포트 (Final_Summary_*.md): (작업 종료 시) 전체 발표 과정을 요약, 평가한 최종 결론 보고서입니다.
  • 순정 이미지 보관 폴더 (Raw_Images_*/): 매 분석 국면마다 캡처된 원본 웹캠 스냅샷 파일(.jpg)들이 안전하게 저장됩니다.

🛑 종료 명령 및 자원 정리

분석을 마치고 싶으실 때는 터미널 창에서 Ctrl + C 키를 누르세요.

  • 실시간 진행 중이던 마지막 분석 루틴까지만 마치고 안전하게 종료됩니다.
  • 마이크 및 카메라 하드웨어 점유율을 완벽히 반환합니다.
  • 실행 중 쌓였던 임시 음성 데이터(temp/ 폴더 내부 파일들)를 깨끗하게 청소합니다.
  • 지금까지의 누적 기록을 바탕으로 [최종 총괄 리포트]를 작성하고 작별을 고합니다.

🔑 필수 사전 요구 사항 (Requirements)

  • .env 파일 내에 GEMINI_API_KEY가 등록되어 있어야 질문 생성 및 번역 기능이 작동합니다.
  • MacBook의 내장 마이크 및 카메라(또는 웹캠) 접근 권한이 터미널에 허용되어 있어야 합니다.
